1. 병합 명령어
merge를 사용하면 HEAD가 가르키는 브랜치에 병합한다.

git merge branch-name => 브랜치를 병합한다.


2. 빨리 감기 병합
master라는 브랜치에서 oldies라는 브랜치를 생성하고 변동사항을 Commit를 했다.
그 이후 master라는 브랜치는 어떠한 변동사항이 존재하지 않았다.
=> 이럴떄 빨리 감기 병합에 해당한다.

oldies라는 브랜치를 master에 브랜치와 합치고 싶다면 master라는 브랜치에서 git branch oldies 명령어를 사용하면 된다.
oldies의 마지막 커밋 포인터를 master와 oldies가 충돌없이 가리키게 된다.


3. 병합 커밋 생성하기
2의 행위를 했지만 두 브랜치가 변경사항이 충돌하지 않는다면 병합 커밋을 생성하고 병합 커밋을 merge를 사용한 브랜치를 가르킨다.

두 개의 마지막 커밋의 내용이 서로 상충되지 않는다면 인위적으로 둘을 잇는 커밋 포인트를 만들고 커밋 포인트를 가지는 대상자는 merge 명령어를 사용한 쪽이 갖는다.

해당 커밋포인트는 기존의 commit 요소와 병합 요소 commit 이력을 모두 갖는다.




4. 병합 충돌
변경사항이 서로 상충하면 개발자가 스스로 일관성을 만들어내야 한다.
=> git이 자동으로 충돌사항을 보여준다. 충돌되는 코드에서 일관성을 만들어낸다.
=> 새로운 커밋포인트에서 git merge을 실행한 브랜치가 소유한다.
=> 충돌사항을 해결한 병합 커밋은 git merge를 실행한 브랜치가 소유하고 병합의 대상자가 된 브랜치는 병합 되기 바로 전 내용을 유지한다.


merge는 브랜치 병합 뿐만 아니라 내용도 병합도 실행한다.
=> stash 같은 것도 병합한다.
