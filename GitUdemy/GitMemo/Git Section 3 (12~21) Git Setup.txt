1. Git 이름 및 이메일 구성하기
Git으로 작업을 하면 Git-log에 git에서 자신이 설정한 이름과 이메일이 나온다.
=> 이는 git은 누가 작업한지를 표현한다는 것이다. 다만 계정등록 같은 개념은 아니다. git에 별칭을 주는 것과 같다.

이름을 설정하는 git 명령어
git config --global user.name "Hong won tae"

git에 설정한 이름을 물어보는 명령어
git config user.name

이메일을 설정하는 git 명령어
git config --global user.email "dnjsxoghd@naver.com"

git에 설정한 이메일을 물어보는 명령어
git config user.email


2. 터미널 명령어-1 (탐색관련 명령어)
ls => list의 축약어로 현재 디렉토리의 컨텐츠들을 나열하는 명령어이다.
ls-a => 숨은 파일까지 보여주는 명령어이다.
start . => 현재 있는 위치를 연다.
clear => 터미널을 clear한다.
ls 폴더명 => 폴더명의 위치의 컨텐츠들을 나열하는 명령어이다.
ls 폴더명/폴더명/폴더명... => 마지막 폴더명의 위치의 컨텐츠들을 나열하는 명령어이다. 메서드 체인과 같은 개념
pwd => 현재 있는 곳의 경로를 반환하는 명령어이다.
cd 파일명 => 파일명이 있는 위치로 이동하는 명령어이다.
cd .. => 전의 위치로 돌아가는 명령어이다.


3. 터미널 명령어-2 (파일 및 폴더 생성 명령어)
touch 파일명.확장자 => 파일을 생성하는 명령어이다.
touch 파일명.확장자 파일명.확장자 파일명.확장자... => 띄어쓰기를 이용해서 여러개를 만들어낼 수 있다.
mkdir 폴더명 => 폴더를 생성하는 명령어이다.


4. 터미널 명령어-3 (파일 및 폴더 삭제)
rm 파일명.확장자 => 자신이 있는 위치에서 해당 파일을 삭제하는 명령어이다. (영구히 삭제된다.)
rm 파일명.확장자 파일명.확장자... => 여러개를 삭제한다.
rm -rf 폴더명 => 자신이 있는 위치에서 해당 폴더를 삭제하는 명령어이다. (영구히 삭제된다.)
rm -rf 폴더명 폴더명... => 여러개의 폴더를 삭제한다.
